# App 007: Keynote Document Export

Track: App Automation
Level: 3
Category: Keynote Automation

## Source

- Site: `iworkautomation`
- Page: `sources/sal/iworkautomation.com/mirror/keynote/document-export.html`

## Goal

Teach export as a core output pattern in Keynote automation.

## What This Lesson Allows

- Connect slide creation to deliverable output
- Show that presentation automation is not complete until it reaches a shareable format
- Reinforce output-oriented thinking across the iWork lessons

## Repo Pairing

- Workflow: `scripts/workflows/keynote/keynote-export-pdf.applescript`

Why this pairing:
It is a direct, visible, and useful export path that matches the preserved source page.

## Suggested Teaching Flow

1. Read the Keynote export page.
2. Explain why export is the handoff point from internal object model to external artifact.
3. Run the export PDF workflow.
4. Compare export to creation and presentation control.

## Practice

- Run `scripts/workflows/keynote/keynote-start-slideshow.applescript` and compare output control with file export.
- Explain which step belongs to authoring and which belongs to publishing.

## Modern Equivalent

- Type: `repo-workflow`

## Outcome

The learner understands that output is a first-class automation concern in Keynote.
