# Foundations 003: AppleScript Features

Track: Foundations
Level: 1
Category: AppleScript Basics

## Source

- Site: `macosxautomation`
- Page: `sources/sal/macosxautomation.com/mirror/applescript/features/index.html`

## Goal

Move from syntax to practical AppleScript feature patterns.

## What This Lesson Allows

- Connect basic script structure to real system tasks
- Introduce state checks, notifications, and app targeting
- Show that AppleScript can coordinate system behavior, not only document apps

## Repo Pairing

- Workflow: `scripts/workflows/system-events/system-events-dark-mode-toggle.applescript`

Why this pairing:
Dark mode is visible and immediate. It demonstrates practical system automation without needing a complex app object model.

## Suggested Teaching Flow

1. Survey the features page for common AppleScript patterns.
2. Run the dark mode toggle.
3. Explain why `System Events` often appears in system-level scripts.
4. Compare app scripting with GUI/system scripting.

## Practice

- Run the script twice and confirm state changes.
- Inspect another `system-events` workflow in this repo.

## Modern Equivalent

- Type: `repo-workflow`
- Target: `scripts/workflows/system-events/system-events-dark-mode-toggle.applescript`

## Outcome

The learner sees that AppleScript features become useful when attached to visible system behaviors.
