Track: Workflow Building
Level: 2
Category: Services And Quick Actions

## Source

- Site: `macosxautomation`

## Goal

Bring the Services curriculum to multi-step utility design.

## What This Lesson Allows

- Frame Services as lightweight applications embedded into context menus and shortcuts
- Show how small utilities can have durable daily value
- Extend the learner from beginner examples into taste and workflow design

## Repo Pairing

- Workflow: `scripts/workflows/finder/finder-tag-orange.applescript`

Why this pairing:
Tagging selected items is exactly the kind of compact, high-frequency utility that belongs in a Service or Quick Action.

## Suggested Teaching Flow

1. Read the fourth Services tutorial page.
2. Discuss what makes a small utility worth preserving.
3. Run the Finder tag workflow.
4. Ask where this should live in the user interface for maximum usefulness.

## Practice

- Compare `scripts/workflows/finder/finder-tag-orange.applescript` with `scripts/workflows/finder/finder-tag-red.applescript`.
- Design a one-step service you would want on the Finder contextual menu.

## Modern Equivalent

- Type: `repo-workflow`
- Target: `scripts/workflows/finder/finder-tag-orange.applescript`

## Outcome

The learner understands that good automation is often small, local, and repeatedly useful.
