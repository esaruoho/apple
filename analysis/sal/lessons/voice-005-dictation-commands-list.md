# Voice 005: Dictation Commands List

Track: Accessibility And Voice
Level: 3
Category: Dictation Commands

## Source

- Site: `dictationcommands`
- Page: `sources/sal/dictationcommands.com/mirror/wayback-20240424/web.archive.org/web/20240424181821/http:/dictationcommands.com/commandslist.html`

## Goal

Preserve the practical command vocabulary that made Dictation Commands usable.

## What This Lesson Allows

- Show that command naming is part of interface design
- Teach consistency and recall as automation concerns
- Connect command vocabulary to real-world usability

## Repo Pairing

- Workflow: `scripts/workflows/homepod/homepod-set-volume.applescript`

Why this pairing:
It behaves like a parameterized spoken command and makes the role of verbs and arguments easy to explain.

## Suggested Teaching Flow

1. Read the commands list page.
2. Identify patterns in command naming and phrasing.
3. Run the HomePod set-volume workflow.
4. Discuss how poor naming makes voice automation unreliable or hard to remember.

## Practice

- Compare `scripts/workflows/homepod/homepod-set-volume.applescript` with `scripts/workflows/homepod/homepod-play-music.applescript`.
- Propose a naming scheme for five spoken commands you would actually use.

## Modern Equivalent

- Type: `shortcut-bridge`
- Target: `scripts/workflows/homepod/homepod-set-volume.applescript`

## Outcome

The learner sees spoken command design as part of automation design, not a cosmetic afterthought.
