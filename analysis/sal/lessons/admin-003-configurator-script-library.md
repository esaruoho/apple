# Admin 003: Configurator Script Library

Track: Admin And Deployment
Level: 4
Category: Configurator Automation

## Source

- Site: `configautomation`
- Page: `sources/sal/configautomation.com/mirror/wayback-20240422/web.archive.org/web/20240422004004/https:/configautomation.com/script-library.html`

## Goal

Show how admin automation depends on reusable script components rather than one-off actions.

## What This Lesson Allows

- Connect operational automation to reusable building blocks
- Teach script libraries as infrastructure, not just examples
- Frame admin work as standardized procedure

## Repo Pairing

- Workflow: `scripts/workflows/automator/automator-list-actions.applescript`

Why this pairing:
It reinforces the concept of discovering and organizing reusable units instead of inventing each workflow from scratch.

## Suggested Teaching Flow

1. Read the Configurator script-library page.
2. Discuss the value of reusable operational components.
3. Run the Automator list-actions workflow.
4. Compare a script library mindset with an ad hoc scripting mindset.

## Practice

- Compare `scripts/workflows/automator/automator-list-actions.applescript` with `scripts/workflows/automator/automator-run-workflow.applescript`.
- Explain what makes a script reusable across many environments.

## Modern Equivalent

- Type: `repo-workflow`
- Target: `scripts/workflows/automator/automator-list-actions.applescript`

## Outcome

The learner sees libraries and action catalogs as core infrastructure for operational automation.
