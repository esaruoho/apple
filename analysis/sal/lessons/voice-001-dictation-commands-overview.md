# Voice 001: Dictation Commands Overview

Track: Accessibility And Voice
Level: 3
Category: Dictation Commands

## Source

- Site: `dictationcommands`
- Page: `sources/sal/dictationcommands.com/mirror/wayback-20240424/web.archive.org/web/20240424181821/http:/dictationcommands.com/index.html`

## Goal

Introduce voice-triggered automation as a first-class accessibility workflow.

## What This Lesson Allows

- Teach that spoken phrases can trigger useful actions
- Connect accessibility and productivity instead of treating them as separate topics
- Preserve the entry-point model from Sal's Dictation Commands material

## Repo Pairing

- Workflow: `scripts/workflows/homepod/homepod-play-music.applescript`

Why this pairing:
It is not Dictation Commands itself, but it preserves the core idea of a voice-adjacent trigger invoking a meaningful automation outcome.

## Suggested Teaching Flow

1. Read the Dictation Commands overview snapshot.
2. Explain why spoken entry points matter for accessibility and speed.
3. Run the HomePod play-music workflow.
4. Map the spoken-phrase idea to a modern Shortcut bridge.

## Practice

- Compare `scripts/workflows/homepod/homepod-play-music.applescript` with `scripts/workflows/homepod/homepod-pause-music.applescript`.
- Write down three spoken commands that would be useful in your own workflow.

## Modern Equivalent

- Type: `shortcut-bridge`
- Target: `scripts/workflows/homepod/homepod-play-music.applescript`

## Outcome

The learner sees voice as a legitimate automation surface, not a novelty layer on top of keyboard-driven scripting.
