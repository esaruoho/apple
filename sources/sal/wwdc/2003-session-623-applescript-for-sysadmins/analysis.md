# WWDC 2003 Session 623 — AppleScript for SysAdmins (Analysis)

**Speaker:** Sal Soghoian (solo, 1:14:11)
**Track:** Enterprise IT
**Note:** Session is unlisted on Apple Developer site. Whisper transcript is the only surviving record.

This is **Sal's solo masterclass** — no co-presenter, no engineer to fall back on. It's the most concentrated dose of pure Sal in the entire 2003 archive, and it contains the most quotable WWSD-relevant material across all four 2003 sessions.

## Sal's opening hooks

> *"AppleScript is the cocaine of programming. One line, and you'll be ours forever. So you have been warned, resist as much as you can."*

> *"How many just kind of like jumped into it because you had to do something and you heard you can do it, but you don't know how it worked and you just pieced it together and you banged your head against the wall 20 times? That's how I learned too. I spent years and years pounding my head against the wall going, 'No, I guess that doesn't work.' And coercion, I thought, was just moving the matter to the other side. I didn't realize it was actually applying to the code in the window."*

> *"My aim with this session is to give you a peek to the inside workings of a scripter's brain and to turn you on to all the hidden secrets that I've developed over the years about how to get up to speed quickly with AppleScript."*

Sal is teaching what experienced AppleScripters know but never write down. The session is *secret knowledge*.

## The big philosophical claim — **"AppleScript is a peer to Aqua"**

> *"AppleScript is a peer to Aqua. Aqua is the graphic user interface to the OS, and AppleScript is the language user interface to the OS. I like to think about it in those terms. There's also another language interface that's really low level, which is the Unix interface that you're familiar with, the terminal and shell and that stuff, too."*

This is a structural framing: **three peer UIs to macOS** — Aqua (graphic), AppleScript (language, high-level), shell (language, low-level). This is the cleanest single-sentence WWSD-grade statement of the philosophy. AppleScript isn't a toy *on top of* Aqua; it's *next to* Aqua.

## The vision sentence (different from 401's vision quote)

> *"Duplicate every file of the startup disk whose name contains Smith Project to the folder named Backup. That's actually a script. It's not just a concept. It's actually a script. And it's the principle behind what AppleScript can do. You take the mind, you rethink the thought, and you write it, and it should work. Most of the time."*

Compositional thinking → English sentence → working script. The 401 vision quote is biographical ("I fell in love in 1992"); this 623 vision sentence is operational ("here's how it actually feels").

## What Sal teaches (the syllabus)

### 1. Five reference types and coercions

The single most useful chunk of the session. Sal enumerates the five ways AppleScript can identify an item:

1. **Nested reference** — `folder documents of folder sal of folder users of startup disk`. Default returned by all scriptable apps. *Cannot be generated by a script.*
2. **Path reference** — `file "Macintosh HD:Users:sal:Documents:cars.pdf"`. Class name + colon-delimited string. Generatable by script (string concatenation).
3. **Alias reference** — `alias "Macintosh HD:Users:sal:Documents:"`. Live reference to existing item; AppleScript tracks it across moves.
4. **POSIX reference** — `/Users/sal/Documents/cars.pdf`. Unix-style; most scriptable apps don't natively accept them.
5. **File URL** — `file://localhost/Users/sal/Documents/`. Sal calls these "little known but useful in extreme situations".

Coercion paths:
- Nested → alias: `... as alias`
- Path → alias: `(file "Macintosh HD:Applications") as alias`
- POSIX → POSIX file → alias: `(POSIX file "/Users/sal/Documents") as alias`
- Alias → POSIX: `POSIX path of <alias>`
- Alias → quoted POSIX: `quoted form of POSIX path of <alias>` (shell-safe with apostrophes auto-added)

> *"If you didn't know that, life is horrible. If you do know that, you're in the air conditioning."*

### 2. Finding items — by index, by property, by query

**By index:**
- Positive index left-to-right: `item 1 of home`, `item 12 of home`
- **Negative index right-to-left**: `item -1 of home` is the last item, `item -2` is second-to-last, etc.
- Range: `items 2 thru 5 of startup disk`, `items 1 thru -1 of startup disk` (== `every item`)

**Descriptive index (conversational):**
- `first item`, `second item`, `22nd item`, `last item`, `middle item`
- *"Conversational. You can write it as terse as you like, or you can write it as verbose as you like."*

**Relative position:**
- `front` / `first` / `back` / `last` / `middle`
- `item before the last item`, `item after the middle item`
- `some item of startup disk` — picks at random

**By property (the query construct):**

Sal lays out the grammar — **positional indicator + target object type + target container + object specifier + target property + comparison operator + property value**:

```applescript
the first document file of home whose name contains "Smith Project"
the first folder whose size is greater than 500000
every document file whose file type is "JPEG"
every document file whose name extension is "pdf"
```

Object specifier can be `whose` or `where its` (synonymous). Comparison operators: contains, does not contain, is in, begins with, ends with, is greater than, is less than, ≥, ≤, isn't, is not.

### 3. Recursion via `entire contents`

> *"How do you get down through a directory, a hierarchy? Well, you go to JavaScript and you write `for if and and`, and then you start doing a recursive loop that recalls itself back up at the top again when you get down to here. No, we don't do that in AppleScript. We're civilized. We're bohemian but civilized."*

```applescript
every document file of the entire contents of folder documents of home ¬
    whose name contains "Smith Project"
```

The Finder/System Events walks the tree for you. You write the query, AppleScript writes the loop.

### 4. The "cleaning and waxing" principle

After running `tell app Finder to get every item of home whose name contains "si"` and then `reveal` on the result:

> *"What we have here is cleaning and waxing in one motion. In one motion I find the stuff, the other one I actually do something with it. In one line of AppleScript, we are able to clean and wax together."*

**Find and act in one line.** Don't separate the query from the action unless you must. This is operational WWSD — the language was designed for it.

### 5. Script Editor tour

Quick orientation for sysadmins:
- Top pane = script
- Bottom pane = description / result / event log (three tabs)
- Description pane is shippable — checkbox at save time → users see a popup with this text when running the script
- Event log shows every command sent + every result returned — debugger by default
- Result pane shows last action's return value

### 6. Dictionary viewer

> *"Applications that are scriptable have scriptable objects. Each of those objects contain certain qualities that make it a scriptable object. And you can examine the entire structure of a scriptable application by asking for its dictionary."*

`File → Open Dictionary → <app>`. Standard Suite = common verbs (close, count, delete, make, open, save). App-specific suite = the app's unique classes + verbs.

Sal walks through QuickTime Player's dictionary: app properties (Boolean: `QuickTime Pro installed`, `show welcome movie`, etc.), Movie class properties (auto-quit, auto-close, controller type, current time, duration, fast start, href).

### 7. Where AppleScript lives

`/Applications/AppleScript/` — Script Editor, Script Menu installer, Folder Actions Setup. *"This is AppleScript's home turf. This is where we live."*

## Sal's repeated emphasis: **free**

> *"All of this is free, by the way. Did I say free? I meant to say free. Did I say free? Free, free, free. You don't have to pay for AppleScript. You don't have to pay for this incredible ability. It's part of every computer that Apple Computer sells."*

Three "free"s in a row. The economic-floor point — *the price of automation on Mac is zero* — is a WWSD-grade principle.

## WWSD-relevant takeaways

- **Peer-to-Aqua framing** — strongest single sentence for AppleScript's structural status in macOS.
- **Conversational > terse** — `22nd item` is as valid as `item 22`. Match the prose, not the syntax minimum.
- **Cleaning-and-waxing** — find + act in one line is the AppleScript signature move.
- **Entire-contents recursion** — language-level recursion, not user-written for-loops. "We're civilized."
- **The five-reference-types matrix** is essential teaching material. Worth promoting from this transcript into a top-level reference.
- **Free, free, free** — the price floor of Mac automation is and always has been zero.

## Reusable for the apple repo

- Sal's coercion table (the five reference types and the as-alias/POSIX file/POSIX path/quoted form patterns) should become a reference card in `dictionaries/` — it's the most-asked question by new AppleScripters and Sal explains it definitively here.
- The `whose ... entire contents` query pattern is widely under-used in `scripts/` — many scripts use manual for-loops that could collapse to a one-liner.
- The "script description checkbox" trick (description tab → tick "show on run") is a UX move worth surfacing in `bin/workflow-gen.py` — auto-generate descriptions from the script's docstring.
- The "library window of pinned apps" idea in Script Editor 2 (covered in 401) is reinforced here — a UX pattern that could be cloned in a Loupedeck launcher.
