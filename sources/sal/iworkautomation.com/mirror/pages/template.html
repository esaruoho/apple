Pages Templates

The template class is a top-level element of the Pages application. It contains no other elements, and has two properties. The choice of template determines the overall look of a document.

The Pages Suite Template Class

template n  : A styled document layout.

elements

contained by application.

properties

id (text, r/o) : The unique identifier for this application element.

name (text, r/o) : The name of the template, as it is displayed in the application’s user interface.

The pages of this topic section detail how to access the properties of all installed templates, all application templates, and all user-created templates. Example scripts detail how to create documents using a specified template.

In Pages, there are two kinds of templates: those that come with the application, and those created and saved by the user.

The id for an application template begins with the term: “Application”

"Application/Blank/Traditional"

The id for a user-created template begins with the term: “User”

"User/C34CF6EC-2BC4-4DE0-9A3B-35BA88F15B2C"

The design of template ids allows for specialized searches, as shown later on this page.

Application|User Templates

A script can use the unique design of the id property to retrieve only templates of a specific type.

The following script retrieves the name of every application template:

Get the Name of All Application Templates
  
01tell application "Pages"
02 set applicationTemplateNames to the name of every template whose id of it begins with "Application/"
03 --> RETURNS: {"Blank", "Blank Landscape", "Note Taking", "Essay", "Term Paper", "School Report", "Visual Report", "Research Paper", "Modern Report", "Project Proposal", "Classic Letter", "Modern Letter", "Business Letter", "Traditional Letter", "Photo Letter", "Elegant Letter", "Bold Type Letter", "Informal Letter", "Classic Resume", "Modern Resume", "Business Resume", "Elegant Resume", "Bold Type Resume", "Informal Resume", "Classic Envelope", "Modern Envelope", "Business Envelope", "Elegant Envelope", "Bold Type Envelope", "Informal Envelope", "Classic Business Cards", "Modern Business Cards", "Business Cards", "Elegant Business Cards", "Bold Type Business Cards", "Informal Business Cards", "For Rent Flyer", "For Sale Flyer", "Tab Flyer", "Real Estate Flyer", "Lost & Found Flyer", "Photo Poster Small", "Photo Poster Big", "Event Poster Small", "Event Poster Big", "Party Poster Small", "Party Poster Big", "School Poster Small", "School Poster Big", "Type Poster Small", "Type Poster Big", "Photo Card Vertical", "Photo Card Horizontal", "Birthday Card", "Real Estate Postcard", "Event Postcard", "Moving Postcard", "Invoice", "Elegant Brochure", "Museum Brochure", "Classic Newsletter", "Journal Newsletter", "Simple Newsletter", "Serif Newsletter"}

Check for Template

A principle of good “defensive” script writing is to never assume anything, always check for resources. Here’s a script that checks to make sure a specific template is intalled:

Check for Template
  
01property thisTemplateName : "Company Newsletter"
02 
03tell application "Pages"
04 activate
05 try
06 -- GET THE TEMPLATE NAMES
07 set the templateNames to the name of every template
08 
09 -- CHECK FOR TEMPLATE
10 if thisTemplateName is not in the templateNames then
11 error "The template “" & thisTemplateName & "” is not installed on this computer."
12 end if
13 
14 on error errorMessage number errorNumber
15 if errorNumber is not -128 then
16 display alert ("ERROR " & errorNumber) message errorMessage
17 end if
18 end try
19end tell
