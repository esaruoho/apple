Images

When it comes to making convincing documents, the old adage of “A picture is worth a thousand words” is sacrosanct. Nothing gets the point across like a compelling image. Not to worry, the AppleScript support in Numbers provides ample controls for creating unforgettable documents.

Scripting Terminology for Images

Documents created by any of the iWork applications, Pages, Numbers, or Keynote, can incorporate a common set of elements, including images, shapes, lines, tables, charts, movies, and audio clips. To AppleScript and the built-in scripting support of the iWork applications, these elements are considered iWork Items, and share a common set of properties, detailed in the iWork Item class from the iWork Suite:

The iWork Suite iWork item Class

iWork item n  [inh. iWork item ] : An item which supports formatting.

elements

contained by slides, document.

properties

height ( integer ) : The height of the iWork item.

locked ( boolean ) : Whether the object is locked.

parent ( iWork container r/o ) : The iWork container containing this iWork item.

position ( point ) : The horizontal and vertical coordinates of the top left point of the iWork item.

width ( integer ) : The width of the iWork item.

responds to

delete, exists

The properties in the iWork item class can be accessed and changed for any of the common iWork elements. For example, a script can get or set the position of a movie, or image, or shape, or any of the common elements.

In addition to sharing a common set of properties with the other iWork elements, each iWork element has its own set of properties, particular to that element. For example, the element-specific properties of an image are detailed in its dictionary listing, found in the iWork Suite:  (⬇ see below ) 

The iWork Suite image Class

image n  [inh. iWork item ] : An image container.

elements

contained by slides, document.

properties

description ( text ) : Text associated with the image, can be read aloud by VoiceOver.

file ( file , r/o) : The image file. This property is a required parameter when used with the make command to create a new image item on a slide. Use the file name property to retrieve the name of the image, or to replace the currently displayed image.

file name ( text or file ) : When used with the get command, the value of this property is the file name of the image file currently displayed in the image container. When this property is used with the set command, the value of the property is a file reference (in HFS alias format) to the image file to be used to replace the currently displayed image.

opacity ( integer ) : The opacity of the image, in percent (0 to 100).

reflection showing ( boolean ) : Is the image displaying a reflection?

reflection value ( integer ) : The percentage of reflection of the image, from 0 (none) to 100 (full).

rotation ( integer ) : The rotation of the image item, in degrees from 0 to 359.

responds to

delete, exists, make

Here’s an example script that creates a new document with a new table overlaid on a decorative background image. It incorporates the use of many of the image properties, as well as those for text items.

Layout with Decorative Background Image
  
01property defaultImageWidth : 792 -- in points
02property defaultTypeSize : 36
03 
04tell application "Numbers"
05 activate
06 
07 -- CREATE THE DOCUMENT
08 set thisDocument to make new document with properties ¬
09 {document template:template "Blank"}
10 
11 tell thisDocument
12 
13 -- GET THE RANDOM IMAGE
14 copy my getRandomDesktopPicture() to {thisImageFileName, thisImageFile}
15 -- trim the file extension (.jpg) from the file name
16 set thisImageName to text 1 thru -5 of thisImageFileName
17 
18 -- ADD THE IMAGE FILE TO THE FIRST SLIDE
19 tell the active sheet
20 -- CLEAR THE CANVAS
21 delete every table
22 
23 -- IMPORT THE DESKTOP IMAGE
24 set thisImage to ¬
25 make new image with properties {file:thisImageFile}
26 tell thisImage
27 set width to defaultImageWidth
28 set position to {0, 0}
29 set opacity to 100
30 set imageWidth to its width
31 set imageHeight to its height
32 copy position of it to {imageHPosition, imageVPositon}
33 end tell
34 
35 -- CREATE A TEXT OVERLAY DISPLAYING IMAGE NAME
36 set thisTitleBox to ¬
37 make new text item with properties {object text:thisImageName}
38 tell thisTitleBox
39 tell its object text
40 set font to "Times New Roman Italic"
41 set size to defaultTypeSize
42 set color of it to {65535, 65535, 65535}
43 end tell
44 set textItemHeight to its height
45 set textItemWidth to its width
46 
47 copy position of it to {textHPosition, textVPosition}
48 set newHPosition to imageHPosition + ((imageWidth div 2) - (textItemWidth div 2))
49 set newVPosition to imageVPositon + 24
50 set position of it to {newHPosition, newVPosition}
51 end tell
52 
53 -- CREATE AN OVERLAY TABLE
54 set thisTable to make new table
55 tell thisTable
56 set the background color of cell range to {65535, 65535, 65535}
57 set tableWidth to its width
58 set tableHeight to its height
59 set newHPosition to imageHPosition + ((imageWidth div 2) - (tableWidth div 2))
60 set newVPosition to imageVPositon + (imageHeight - tableHeight - 36)
61 set position of it to {newHPosition, newVPosition}
62 end tell
63 end tell
64 end tell
65end tell
66 
67on getRandomDesktopPicture()
68 -- GET A RANDOM IMAGE FROM THE DESKTOP pictures folder
69 set desktopPicturesFolder to (path to desktop pictures folder)
70 set the imageFileNames to every paragraph of ¬
71 (do shell script "ls " & quoted form of POSIX path of desktopPicturesFolder)
72 --> RETURNS: {"Abstract.jpg", "Antelope Canyon.jpg", "Bahamas Aerial.jpg", etc.}
73 set thisImageFileName to some item of the imageFileNames
74 --> RETURNS: "Mountain Range.jpg"
75 set thisImageFile to ((desktopPicturesFolder as string) & thisImageFileName) as alias
76 --> RETURNS: alias "Macintosh HD:Library:Desktop Pictures:Mountain Range.jpg"
77 return {thisImageFileName, thisImageFile}
78end getRandomDesktopPicture
