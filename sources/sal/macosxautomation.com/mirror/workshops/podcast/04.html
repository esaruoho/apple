Play and Render the First Narration Clip

Now that the services have been created and installed, we can use them to create the rendered narration audio clips that are added to the GarageBand slideshow project.

Preview the Narration

The first step in creating a rendered track is to preview the the narration with the voice you want to use for rendering.

DO THIS ►Select the text for the first narration clip by clicking anywhere within the text area below.

01-MALE:

DO THIS ►With the text selected, type the keystroke combination you assigned for reading the text with a male voice: Control-2 (⌃2)

The selected text will be read using the chosen male voice.

Render the Narration

The next step is to render the selected text into an audio file and open the rendered file in QuickTime Player application.

DO THIS ►With the text selected, type the keystroke combination you assigned for rendering the text with a male voice: Control-Option-2 (⌃⌥2)

The selected text will be rendered into an audio file and opened in the QuickTime Player application.

Play the Rendered Narration

DO THIS ► Play the audio clip to confirm that the text was rendered properly.

Congratulations, using the Automator servics you created, you easily turned selected text into a narrated audio file! Next, you’ll learn how to add the audio clip to the GarageBand project template.

WORKSHOP LEADER NOTES

Segment Overview

In this segment, the attendees use the new services they created to preview selected text and render it to an audio file.

Talking Points

  1. Once services are created, they become useful tools, available whenever you need them.
  2. Services can be activated not only via assigned keystroes but also through the use of application and contextual menus.
  3. Show how services can also be accessed by Control-clicking in the text seletion to call up the application’s contextual menu containing the created services. (view)
  4. Show how services can also be accessed from an applicaation’s Services menu. (view)

Mention that the purpose of auditioning the selected text with a voice is to ensure that the text is written correctly and that it contains realistic pauses and inflections. Some of the narration dialog will contain phonetic spelling to force the speech software to speak the words in specific ways.

Also, narration text may contain commas, semi-colons, and speech tags to add pauses at important points in the text. For example, the tag [[slnc 500]] is not spoken by the speech engine, but is instead read as an instruction to pause one-half second (500 milliseconds) before continuing to speak the selected text.
