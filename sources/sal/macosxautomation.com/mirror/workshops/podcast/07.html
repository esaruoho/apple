Workshop Session Summary

Congratulations. You’ve completed the process of creating, setting up, and using custom automation tools for OS X.

Specifically:

  1. Using Automator, you created service workflows that use text selected in documents or windows as input for the built-in text-to-speech abilities of OS X.
  2. In addition, you enabled these services to be easily accessible throughout the system by assigning them keyboard shortcuts.
  3. And you incorporated the use of the new services in the process of constructing a media presentation.

In short, you’ve discovered that the power and abilities of automation in OS X is available for you use. If you find yourself thinking “I wonder if there’s a better way to do this?” then it’s time to consider using automation.

WORKSHOP LEADER NOTES

Segment Overview

Engage the attendees in conversation about the workshop, lesson learned, ideas for other uses.

Talking Points

  1. Services created with Automator can be adapted, expanded, simplified, etc. To edit an existing service, choose “Open” from the Automator “File” menu, and then select “Service” from the “Type” popup menu in the forthcoming dialog. The services installed in the user Services folder will be selectable for editing.
  2. Services created with Automator can be shared with others. Mention how the service workflow files, located in the Home > Library > Services folder can be copied to other computers, and then installed by double-clicking the workflow files in the Finder. (installation details here)
  3. The services created for this workshop can be used in other ways aside from creating narration tracks for podcasts. The same services can be used to add voice to Keynote presentations, and even for creating movies, such as this short movie about Automator. Discuss possible use scenarios with the attendees.
  4. In addition to the voices that come with OS X, there are very high-quality voices available from the 3rd-party deelopers linked below.
  5. Written text can include special embedded Speech Synthesis Commands to make text rendered to audio by the computer, sound more natural. For detailed information on these special codes, visit the Apple developer website pages for Speech Synthesis
